API Documentation

Welcome to the BlackRoad OS API. This API allows you to manage users, subscriptions, and integrate with our platform.

Base URL: https://api.blackroad.io

Authentication

The API uses JWT (JSON Web Tokens) for authentication. Include the token in the Authorization header:

Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

Getting a Token

You can obtain a token by registering or logging in:

// Register a new user
POST /api/auth/register
{
  "email": "user@example.com",
  "password": "yourpassword",
  "name": "Your Name"
}

// Login
POST /api/auth/login
{
  "email": "user@example.com",
  "password": "yourpassword"
}

Token Refresh

Access tokens expire after 1 hour. Use the refresh token to get a new access token:

POST /api/auth/refresh
Authorization: Bearer <refresh_token>

Error Handling

The API returns standard HTTP status codes. Error responses include a message:

{
  "error": "Description of what went wrong"
}
Code Meaning
200 Success
201 Created
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid or missing token
403 Forbidden - Insufficient permissions
404 Not Found
429 Rate Limit Exceeded
500 Server Error

Stripe Webhooks

Configure your Stripe webhook to point to:

POST https://api.blackroad.io/webhook/stripe

Supported events:

  • checkout.session.completed
  • customer.subscription.created
  • customer.subscription.updated
  • customer.subscription.deleted
  • invoice.payment_succeeded
  • invoice.payment_failed
